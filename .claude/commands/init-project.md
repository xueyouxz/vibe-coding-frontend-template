根据用户提供的项目概览、初始需求，或“基于该模板重构一个项目/实现一个系统”的描述，对当前模板进行项目初始化。

目标：
1. 将模板默认文案替换为真实项目内容
2. 移除首页中的模板介绍性内容，改为项目自己的首页或占位内容
3. 更新项目基础元信息，使仓库从模板切换为具体项目
4. 完成后执行代码检查
5. 输出初始 git 提交命令和项目文件夹命名建议（英文，多选）

执行要求：
1. 先从用户输入中提炼以下信息：
   - 项目名称
   - 项目定位/一句话描述
   - 核心功能或业务域
   - 是否需要基于现有模板结构继续扩展，还是先做模板去品牌化重构
2. 按需修改以下内容：
   - `src/features/home/` 中与模板介绍相关的首页文案或展示模块
   - `index.html` 中的 title
   - `package.json` 中的 `name`、`description`
   - `README.md` 中的模板描述
   - `.env.example` 中与项目名相关的示例值（如适用）
3. 保持现有 feature-based 架构，不要破坏目录约定
4. 若用户需求还不足以生成真实业务首页，则至少：
   - 移除“模板说明”语义
   - 改成该项目的名称、定位和下一步建设方向
5. 不要无意义批量修改与模板初始化无关的文件

检查要求：
1. 运行 `pnpm lint`
2. 如本次修改影响构建入口或元信息，额外运行 `pnpm build`
3. 若检查失败，先修复再结束

最终输出必须包含：
1. 本次初始化修改了哪些关键内容
2. 检查结果
3. 建议执行的初始提交命令，例如：
   - `git add -A`
   - `git commit -m "chore: initialize project"`
4. 3-5 个英文项目文件夹命名建议，命名风格需适合真实仓库名，例如：
   - `acme-admin`
   - `pulse-dashboard`
   - `nova-portal`

如果用户没有提供足够信息，优先基于其描述做最小可用初始化，不要停在提问阶段。
